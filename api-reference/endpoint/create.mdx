<RequestExample>

```bash cURL
curl -X POST https://api.company.com/v1/scr/consultation \
  -H "Authorization: Bearer your_jwt_token" \
  -H "Content-Type: application/json" \
  -d '{
    "document": "12345678901",
    "document_type": "cpf",
    "include_score": true,
    "include_history": false
  }'
```

```javascript JavaScript
const response = await fetch("https://api.company.com/v1/scr/consultation", {
  method: "POST",
  headers: {
    Authorization: "Bearer your_jwt_token",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({

    document_type: "cpf",

const scrData = await response.json();
console.log(scrData);
```

```python Python
import requests

url = "https://api.company.com/v1/scr/consultation"
headers = {
    "Authorization": "Bearer your_jwt_token",
    "Content-Type": "application/json"
}
payload = {
    "document": "12345678901",
    "document_type": "cpf",
    "include_score": True,
    "include_history": False
}

response = requests.post(url, json=payload, headers=headers)
scr_data = response.json()
```

```php PHP
<?php
$url = 'https://api.company.com/v1/scr/consultation';
$data = [
    'document' => '12345678901',
    'document_type' => 'cpf',

    'include_history' => false

$options = [

        'method' => 'POST',
        'content' => json_encode($data)

$context = stream_context_create($options);
$result = file_get_contents($url, false, $context);

?>
```

## Request Parameters

<ParamField body="document" type="string" required>
  The document number to query (CPF or CNPJ)
</ParamField>

<ParamField body="document_type" type="string" required>
  Type of document being queried
  <Expandable title="Available options">
    - `cpf` - Individual taxpayer registry (11 digits) - `cnpj` - Company
    taxpayer registry (14 digits)
  </Expandable>
</ParamField>

<ParamField body="include_score" type="boolean" default="true">
  Whether to include credit score information in the response
</ParamField>

<ParamField body="include_history" type="boolean" default="false">
  Whether to include historical debt information (additional charges may apply)
</ParamField>

<ParamField body="callback_url" type="string">
  Optional webhook URL to receive the result asynchronously
</ParamField>

## Response Fields

<ResponseField name="request_id" type="string">
  Unique identifier for this consultation request
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 timestamp when the consultation was processed
</ResponseField>

<ResponseField name="document" type="string">
  The queried document number (may be masked for privacy)
</ResponseField>

<ResponseField name="document_type" type="string">
  Type of document that was queried
</ResponseField>

<ResponseField name="person" type="object">
  Personal information of the document holder
  <Expandable title="person properties">
    <ResponseField name="person.name" type="string">
      Full name of the person
    </ResponseField>
    <ResponseField name="person.birth_date" type="string">
      Birth date in YYYY-MM-DD format
    </ResponseField>
    <ResponseField name="person.mother_name" type="string">
      Mother's full name
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="credit_score" type="object">
  Credit score information (if include_score is true)
  <Expandable title="credit_score properties">
    <ResponseField name="credit_score.score" type="integer">
      Numeric credit score (0-1000 scale)
    </ResponseField>
    <ResponseField name="credit_score.classification" type="string">
      Score classification: EXCELLENT, GOOD, FAIR, POOR, BAD
    </ResponseField>
    <ResponseField name="credit_score.last_updated" type="string">
      When the score was last calculated
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="financial_summary" type="object">
  Summary of financial obligations
  <Expandable title="financial_summary properties">
    <ResponseField name="financial_summary.total_debt" type="number">
      Total amount of registered debts
    </ResponseField>
    <ResponseField name="financial_summary.overdue_debt" type="number">
      Amount of overdue debts
    </ResponseField>
    <ResponseField name="financial_summary.number_of_creditors" type="integer">
      Number of different creditors
    </ResponseField>
    <ResponseField name="financial_summary.oldest_debt_date" type="string">
      Date of the oldest registered debt
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="debts" type="array">
  Array of debt records
  <Expandable title="debt object properties">
    <ResponseField name="debts[].creditor" type="string">
      Name of the creditor institution
    </ResponseField>
    <ResponseField name="debts[].debt_type" type="string">
      Type of debt (FINANCING, CREDIT_CARD, LOAN, etc.)
    </ResponseField>
    <ResponseField name="debts[].amount" type="number">
      Debt amount in Brazilian Real (BRL)
    </ResponseField>
    <ResponseField name="debts[].status" type="string">
      Current status: ACTIVE, PAID, OVERDUE, CANCELLED
    </ResponseField>
    <ResponseField name="debts[].registration_date" type="string">
      When the debt was registered in SCR
    </ResponseField>
    <ResponseField name="debts[].due_date" type="string">
      Due date (null for revolving credit)
    </ResponseField>
  </Expandable>
</ResponseField>

## Document Validation

The API validates document formats according to Brazilian standards:

<CardGroup cols={2}>
  <Card title="CPF Format" icon="user">
    - 11 digits only (numbers) - Valid check digits - Not a known invalid
    sequence
  </Card>
  <Card title="CNPJ Format" icon="building">
    - 14 digits only (numbers) - Valid check digits - Active company
    registration
  </Card>
</CardGroup>

## Rate Limits

<Info>
  **Standard Plan**: 100 requests per minute, 5,000 requests per day **Premium
  Plan**: 500 requests per minute, 50,000 requests per day
</Info>

## Data Privacy

All SCR consultations are logged for compliance purposes. The queried document holder will be notified of the consultation according to Brazilian data protection regulations (LGPD).

<Warning>
  Only query documents for which you have explicit consent or legitimate
  business interest.
</Warning>
