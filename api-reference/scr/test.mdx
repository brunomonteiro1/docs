---
title: "Modalidades"
mode: "custom"
description: "As modalidades de operação representam a classificação detalhada das operações de crédito registradas no SCR Bacen. Cada operação é identificada por um código único (modalidade) e está vinculada a um grupo de modalidades mais amplo, conforme estrutura definida pelo Banco Central."
---

<RequestExample>

```bash JSON
{
  "tipo": "A VENCER",
  "modalidade": "0101",
  "dominio": "Adiantamentos a depositantes",
  "subdominio": "Adiantamentos a depositantes",
  "valorVencimento": 0
},
```

</RequestExample>

<RequestExample>

```bash JSON
{
    "tipo": "A VENCER",
     "modalidade": "02XX",
     "dominio": "Empréstimos",
     "subdominio": "Cheque especial e conta garantida",
     "valorVencimento": 0
},
```

</RequestExample>

<sub>A estrutura é composta por:</sub>

- **<sub>Categorias:</sub>**<sub> representam agrupamentos amplos de operações com características similares, como </sub>_<sub>Empréstimos</sub>_<sub>, </sub>_<sub>Financiamentos</sub>_<sub>, </sub>_<sub>Direitos Creditórios</sub>_<sub>, entre outras.</sub>
- **<sub>Modalidades:</sub>**<sub> são operações específicas dentro de cada categoria, que definem com maior precisão o tipo de contrato informado ao SCR. Exemplos incluem: </sub>_<sub>Crédito Pessoal Consignado</sub>_<sub>, </sub>_<sub>Crédito Rotativo</sub>_<sub>, </sub>_<sub>Vendor</sub>_<sub>, </sub>_<sub>Financiamento de Veículos</sub>_<sub>, entre outros.</sub>

<sub>Cada código </sub>`Modalidade`<sub>deve ser interpretado de forma consistente com sua descrição oficial e utilizado conforme a natureza da operação e o contexto do relatório, conforme detalhado a seguir:</sub>
